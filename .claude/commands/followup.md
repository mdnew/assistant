# Draft Follow-Up

Draft a follow-up message for: $ARGUMENTS

## Instructions

1. **Gather context**:
   - Look up the person in `stakeholders/roster.md`
   - Find relevant meeting notes in `meetings/notes/`
   - Check deal status in `deals/pipeline.md` if applicable
   - Check any pending action items in `meetings/actions/`

2. **Identify the purpose** of the follow-up:
   - Recapping what was discussed
   - Moving something forward / requesting a next step
   - Sharing a resource or proposal
   - Checking in on a stale deal or conversation

3. **Draft the message**:
   - Short. BD follow-ups should be under 150 words unless there's a lot to share.
   - Specific subject line (not "Following up")
   - Reference something personal/specific from the conversation
   - One clear ask or next step
   - Friendly but not sycophantic

4. **Present the draft** and ask if I want any changes before sending

5. **Optionally send** via Gmail MCP (`GMAIL_SEND_EMAIL`) if I confirm

After I confirm, also:
- Log any action items in `meetings/actions/`
- Update the deal stage if relevant
