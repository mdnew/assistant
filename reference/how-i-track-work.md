# How I Track Work

## daily standup.txt — Today + this week

- **Purpose**: What I'm doing today and goals for this week.
- **Where**: Repo root; iCloud-synced so I can edit from any device.
- **Content**: Current week's standup (Mon–Fri sections), today's tasks, this week's goals, blockers.
- **Archive**: Each day a copy is saved to `standups/YYYY-MM-DD.md` so there's a daily record. When I say "log standup" or "post standup", the assistant reads `daily standup.txt`, writes that day's snapshot to `standups/YYYY-MM-DD.md`, and can clear the logged sections (or leave them).

## to-do.txt — Longer-term (next 3 months)

- **Purpose**: Longer-term goals and project outcomes for the next ~3 months.
- **Where**: Repo root.
- **Content**: Goals by project/client (KQED, Prism, Tanita, Uptech BD, etc.) and the key outcomes we're working toward. Not day-by-day; use when planning or reviewing what we're working toward.

## Tracked elsewhere

- **LinkedIn / content posts** — Scheduled posts and content are tracked in a different repo. Don't duplicate that tracking here; this repo is for deals, meetings, standups, and project goals.

## Summary

| File | Time horizon | Use for |
|------|--------------|--------|
| `daily standup.txt` | Today + this week | Day-to-day doings, weekly goals, blockers; copy saved daily to `standups/` |
| `to-do.txt` | Next ~3 months | Longer-term goals and outcomes by project |
